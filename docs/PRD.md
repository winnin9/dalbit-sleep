## 1. 개요

달빛수면은 수면 퀄리티를 별점으로 기록하고, 기상 후 달빛 조각을 모아 보름달을 완성하는 수면 습관 앱이다. 토스(Toss) 앱 내의 AppInToss 미니앱으로 동작하며, 토스의 자동 로그인을 활용해 진입 장벽을 없앴다. 기록 습관 형성을 돕고, 보름달 완성 시 토스 포인트를 직접 지급하여 지속적인 동기를 부여한다.

### 핵심 루프

```
토스 앱에서 진입 (자동 로그인 처리)
        ↓
어젯밤 수면 별점(1~5) + 메모(선택) 기록
        ↓
수면의 질과 관계없이 기록 자체에 대한 보상 (달빛 조각 적립)
        ↓
달빛 조각 달성 → 보름달 완성 (첫 회는 빠른 성취 제공)
        ↓
토스 포인트 받기 (AppInToss 프로모션 API 사용)
```

## 2. 문제 정의

### 타겟 사용자

**40~50대 주부** (주 타겟)

| 특성 | 내용 |
|------|------|
| 수면 고민 | 갱년기·스트레스로 인한 수면 장애 |
| 인식 | "잠이 보약" — 수면 중요성을 몸으로 아는 세대 |
| 동기 | 건강 관리에 관심, 꾸준한 습관 형성 욕구 |
| 디지털 수준 | 별도 앱 설치나 회원가입을 귀찮아함 (AppInToss가 최적) |

### 해결하는 문제 및 기존 안의 개선
- **기존의 문제**: 수면의 질이 좋은 날(4~5점)에만 조각을 주면, 진성 불면증 환자는 달빛을 모으지 못해 상실감을 느끼고 이탈하거나 포인트를 위해 거짓 기록을 하게 됨. 
- **개선안**: 잘 잔 날은 물론, 못 잔 날(1~3점)에도 '위로의 별조각'을 지급해 꾸준한 기록 행위 자체를 보상.
- **초기 이탈 문제**: 첫 보상을 받기까지 너무 긴 시간(보름 남짓)이 걸리면 동기가 시듦. 첫 번째 보름달은 매우 적은 조각으로 완성되도록 허들을 낮춰 빠른 성취감(Aha-moment)을 제공.

## 3. 목표 및 성공 지표

### 목표

1. 기상 후 수면 기록을 토스 앱 내 생활 습관으로 정착시킨다.
2. 월 2~3회 자연스러운 노출로 리텐션을 확보하고 포인트 지급/광고 보상 구조를 완성한다.

| 월간 보름달 완성율 | 활성 유저의 50% 이상 |
| 평균 기록 연속일 | 10일 이상 |

## 4. 달빛 조각 적립 모델 (개선됨)

### 적립 규칙 (기록 자체를 보상)

| 수면 별점 | 달빛 조각 | 설명 |
|-----------|-----------|------|
| 5점 (매우 잘 잠) | +2조각 | "푹 주무셨네요! 밝은 달빛을 드릴게요" |
| 4점 (잘 잠) | +2조각 | "좋은 수면이에요! 달빛이 모였어요" |
| 1~3점 (못 잠) | +1조각 | "피곤하시겠어요. 위로의 달빛 조각을 드릴게요" |

### 보름달 완성 조건

- **튜토리얼 (최초 1회)**: **5조각** (빠른 성취를 위함 - 최단 3일 내 완성)
- **일반 (이후)**: **15조각** 적립 시 보름달 완성
- 완성 트리거 달성 시, 포인트 전환 가능 액션 활성화

### 달 시각화

```
초승달(0) → 반달(진행도 50%) → 보름달(진행도 100%)
```

- 조각이 쌓일수록 달이 차오르는 애니메이션으로 성취감 제공.

## 5. 기능 명세 (Feature Specification)

### 5.1 수면 기록

- 앱 실행 시 당일 기록이 없으면 자동으로 기록 화면(바텀시트 권장) 호출
- **입력 항목**: 수면 별점 (필수, 1~5점), 오늘의 한마디 (선택)
- 하루 1회 한정 기록. 당일 수정 불필요 시 소급 불가.

### 5.2 달빛 달력 (메인 화면)

- **상단**: 현재 달 시각화 
- **중단**: 이번 달 조각 수 / 진행 바 (예: 7/15 조각 표시)
- **하단**: 최근 수면 기록 캘린더 (적립된 조각/별점 표시)

### 5.3 보름달 완성 & 보상 (AppInToss 연동)

#### 완성 화면 및 포인트 전환
1. 조각을 모두 모으면 보름달 축하 애니메이션 노출.
2. **[토스 포인트 받기]** 버튼 클릭 시, AppInToss `grantPromotionReward` API를 호출하여 즉시 토스 포인트를 지급. 
  - (선택/수익성 고려) 포인트 지급 직전 광고 API `ad.showRewardedAd()` 추가 연동 가능.
3. "토스 포인트가 도착했어요!" 완료 처리 후 달 조각 초기화.

### 5.4 수면 통계 확인

- 하단 또는 탭에 위치
- 연속 기록일, 주간 평균 별점, 별점 추이 등 기기 내 저장된 기록을 바탕으로 보여줌.

## 6. AppInToss 기술 제약 및 정책 (Constraints & Policies)

앱인토스 미니앱 가이드(`.claude/skills/appintoss-launch/SKILL.md`)에 따라 아래의 기술 스펙을 준수한다.

| 항목 | 내용 및 해결책 |
|------|----------------|
| 인증 및 식별 | 토스 앱 내포에 따른 자동 로그인. 별도 가입 절차 불필요.<br>유저 식별은 기기 식별/익명 정보 추출을 위한 `getAnonymousKey()` 활용. (또는 서버 도입 시 `appLogin()`) |
| 데이터 저장 | 자체 서버 구축(v1.x)을 생략. 대신 AppInToss `Toss Storage API`(`useStorage` 훅)를 활용하여 메인 상태(조각 수, 이전 기록)를 토스 앱 환경에 안전하게 저장. (로컬스토리지 대비 안전하고 영구적 보존성에 근접) |
| 포인트 보상 | 서버 연동 없이 AppInToss `grantPromotionReward` API를 클라이언트에서 직접 호출. 비즈 월렛에 예산을 충전하여 토스 포인트 즉시 지급. |
| UI / 기본 구동 | `granite.config.ts` 설정에 입각. React 기반. 컴포넌트나 색상은 `@toss/tds-mobile` 패키지와 가급적 조합하여 네이티브앱의 룩앤필 제공. |
| 광고(수익) | `ad` 객체 활용 (AppInToss web-bridge 리워드형/전면형 광고 API 연동, 필요 시 반영). |
